## 🎯 技术原理

### 什么是 IntersectionObserver？

IntersectionObserver 是一个浏览器 API，用于**异步观察**元素与视口（或其他指定根元素）的交叉状态。

### 核心优势

```javascript
// 传统方案：监听 scroll 事件（性能差）
window.addEventListener('scroll', () => {
  images.forEach(img => {
    if (isInViewport(img)) {
      img.src = img.dataset.src
    }
  })
})
// ❌ 问题：scroll 事件触发频率极高，导致性能问题
// ❌ 问题：需要手动计算元素位置，容易出现 bug
// ❌ 问题：滚动时持续执行，影响用户体验

// ✅ IntersectionObserver 方案
const observer = new IntersectionObserver(callback, options)
observer.observe(targetElement)
// ✅ 浏览器自动优化，只在元素可见性变化时触发
// ✅ 运行在主线程之外，不影响页面性能
// ✅ 支持根元素、阈值配置，灵活度高
```

## 💻 原生实现

### 基本用法

```javascript
// 1. 创建观察器
const observer = new IntersectionObserver((entries, observer) => {
  entries.forEach(entry => {
    // entry.isIntersecting: 元素是否进入可视区域
    // entry.boundingClientRect: 元素边界信息
    // entry.intersectionRatio: 交叉比例（0-1）
    
    if (entry.isIntersecting) {
      const img = entry.target
      img.src = img.dataset.src
      observer.unobserve(img) // 停止观察已加载的图片
    }
  })
}, {
  root: null,              // null = 视口
  rootMargin: '0px',      // 观察区域扩展（可让图片提前加载）
  threshold: 0.1           // 10% 可见时触发
})

// 2. 开始观察
document.querySelectorAll('img[data-src]').forEach(img => {
  observer.observe(img)
})

// 3. 停止观察
observer.disconnect()
```

### 完整示例

```javascript
class LazyLoad {
  constructor(options = {}) {
    this.options = {
      root: null,
      rootMargin: '50px 0px',
      threshold: 0,
      onLoad: null,
      onError: null,
      ...options
    }
    
    this.images = []
    this.observer = null
    
    this.init()
  }
  
  init() {
    // 创建观察器实例
    this.observer = new IntersectionObserver(
      this.handleIntersection.bind(this),
      this.options
    )
    
    // 观察所有待懒加载的图片
    this.observeImages()
  }
  
  handleIntersection(entries) {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        this.loadImage(entry.target)
        this.observer.unobserve(entry.target)
      }
    })
  }
  
  loadImage(img) {
    const src = img.dataset.src
    
    const tempImage = new Image()
    
    tempImage.onload = () => {
      img.src = src
      img.classList.add('loaded')
      this.options.onLoad?.(img)
    }
    
    tempImage.onerror = () => {
      img.classList.add('error')
      this.options.onError?.(img)
    }
    
    tempImage.src = src
  }
  
  observeImages() {
    document.querySelectorAll('img[data-src]').forEach(img => {
      this.images.push(img)
      this.observer.observe(img)
    })
  }
  
  destroy() {
    this.observer.disconnect()
    this.images = []
  }
}

// 使用
const lazyLoad = new LazyLoad({
  rootMargin: '100px 0px',
  onLoad: (img) => console.log('图片加载成功:', img.src),
  onError: (img) => console.error('图片加载失败:', img.src)
})
```

## 🎨 Vue 3 封装

### 方式一：自定义指令（推荐）

```javascript
// src/directives/lazyload.js
export default {
  mounted(el, binding) {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          el.src = binding.value
          observer.unobserve(el)
        }
      },
      { rootMargin: '50px 0px' }
    )
    
    el._observer = observer
    observer.observe(el)
  },
  
  unmounted(el) {
    el._observer?.disconnect()
  }
}
```

**使用：**
```vue
<img v-lazyload="imageUrl" />

// main.js 中注册
app.directive('lazyload', lazyloadDirective)
```

### 方式二：组合式函数（Hooks）

```javascript
// src/hooks/useLazyLoad.js
import { ref, onMounted, onUnmounted } from 'vue'

export function useLazyLoad(options = {}) {
  const imageRef = ref(null)
  const isLoaded = ref(false)
  const isInView = ref(false)
  let observer = null
  
  const defaultOptions = {
    rootMargin: '100px 0px',
    threshold: 0,
    ...options
  }
  
  onMounted(() => {
    if (!imageRef.value) return
    
    observer = new IntersectionObserver((entries) => {
      const entry = entries[0]
      if (entry.isIntersecting) {
        isInView.value = true
        observer.unobserve(imageRef.value)
      }
    }, defaultOptions)
    
    observer.observe(imageRef.value)
  })
  
  onUnmounted(() => {
    observer?.disconnect()
  })
  
  return { imageRef, isLoaded, isInView }
}
```

**使用：**
```vue
<script setup>
import { useLazyLoad } from '@/hooks/useLazyLoad'

const { imageRef, isLoaded, isInView } = useLazyLoad({
  rootMargin: '200px 0px'
})

const loadImage = () => {
  // 自定义加载逻辑
}
</script>

<template>
  <div ref="imageRef" v-if="isInView">
    <img :src="imageUrl" @load="isLoaded = true" />
  </div>
</template>
```

## ⚡ 性能优化技巧

### 1. 提前加载（Progressive Loading）

```javascript
// 在元素进入视口前 100px 开始加载
const options = {
  rootMargin: '100px 0px'  // 提前加载
}

// 面试点：解释 rootMargin 的作用
// rootMargin: '50px 0px' 表示在计算交叉时，将根元素的边界向外扩展 50px
// 这样元素距离视口还有 50px 时就会触发加载
```

### 2. 占位图（Placeholder）

```css
/* 模糊占位图 */
.lazy-placeholder {
  filter: blur(10px);
  transform: scale(1.1);
  transition: filter 0.3s;
}

/* 加载完成 */
.lazy-loaded {
  filter: blur(0);
  transform: scale(1);
}
```

### 3. 渐进式图片（Progressive JPEG）

```javascript
// 先加载低质量图，再加载高清图
const lowQualitySrc = `${src}?param=50y50`
const highQualitySrc = `${src}?param=300y300`

img.src = lowQualitySrc  // 先加载小图
img.onload = () => {
  img.src = highQualitySrc  // 再加载大图
}
```

### 4. 懒加载 + 虚拟列表

```vue
<!-- 当列表很长时，结合虚拟滚动 -->
<virtual-list :data="songs">
  <template #default="{ item }">
    <LazyImage :src="item.cover" />
  </template>
</virtual-list>
```

### 5. 懒加载 + 预加载策略

```javascript
// 当前图片加载时，预加载下一批图片
const preloadNextBatch = (currentIndex) => {
  const batch = images.slice(currentIndex + 1, currentIndex + 4)
  batch.forEach(src => {
    const img = new Image()
    img.src = src
  })
}
```

## 🎤 面试话术

### Q1: 图片懒加载的实现原理是什么？

**回答要点：**
> "图片懒加载的核心是**延迟加载**，只有当图片进入可视区域时才加载真实资源。
> 
> 我使用 IntersectionObserver API 实现，它有以下几个优势：
> 1. **性能优化**：传统方案监听 scroll 事件，触发频率高，性能差。IntersectionObserver 由浏览器优化，只在元素可见性变化时触发
> 2. **异步执行**：观察器的回调函数在主线程之外执行，不阻塞页面渲染
> 3. **精确控制**：支持 rootMargin 和 threshold 配置，可以精确控制触发时机"

**代码示例：**
```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target
      img.src = img.dataset.src
      observer.unobserve(img)  // 停止观察，提升性能
    }
  })
}, { rootMargin: '100px 0px' })  // 提前 100px 加载
```

### Q2: 如何优化首屏加载速度？

**回答要点：**
> "首屏优化是前端性能提升的关键环节，我通过以下方式优化：
> 
> **1. 图片懒加载**
> - 使用 IntersectionObserver 延迟加载非首屏图片
> - 通过 rootMargin: '100px' 提前 100px 开始加载，保证用户体验
> 
> **2. 骨架屏 + 渐进式加载**
> - 加载过程中显示骨架屏占位
> - 图片使用模糊到清晰的渐进式过渡
> 
> **3. 资源压缩**
> - 图片使用 WebP 格式，体积减少 30-50%
> - 使用 CDN 分发，配合 HTTP/2 多路复用
> 
> **4. 缓存策略**
> - 配置 Cache-Control，利用浏览器缓存
> - 首次加载后，图片命中强缓存，后续访问秒开"

**实际项目中的数据：**
> "在我的音乐播放器项目中，应用懒加载后：
> - 首屏加载时间从 **3.2s 降至 1.5s**
> - 图片请求数量减少 **60%**
> - 用户可交互时间（TTI）提升 **40%**"

### Q4: 懒加载的常见问题及解决方案？

**回答：**
> **问题1：图片闪烁**
> - **原因**：占位图和真实图片尺寸不一致
> - **解决**：统一占位图尺寸，设置 object-fit: cover
> 
> **问题2：SEO 不友好**
> - **原因**：img 标签没有 src
> - **解决**：使用 data-src 存储真实地址，保持 alt 文本完整
> 
> **问题3：首屏图片也需要懒加载**
> - **原因**：首屏图片延迟加载会影响 FCP
> - **解决**：首屏图片直接加载，非首屏使用懒加载

### Q: 如何判断图片是否加载成功？

```javascript
img.onload = () => console.log('加载成功')
img.onerror = () => console.error('加载失败')

// 也可以使用 Promise 封装
const loadImage = (src) => {
  return new Promise((resolve, reject) => {
    const img = new Image()
    img.onload = () => resolve(img)
    img.onerror = reject
    img.src = src
  })
}
```

### Q: 如何处理大量图片的懒加载？

```javascript
// 分批观察，避免一次观察过多元素
const batchObserve = (images, batchSize = 10) => {
  let index = 0
  
  const observeBatch = () => {
    const batch = images.slice(index, index + batchSize)
    batch.forEach(img => observer.observe(img))
    index += batchSize
    
    if (index < images.length) {
      requestIdleCallback(observeBatch) // 利用空闲时间
    }
  }
  
  observeBatch()
}
```
